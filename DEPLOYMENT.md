# 🚀 Center El-Sayad — Deployment Guide\n\nThis guide covers deploying the full stack to production:\n- **Backend API** (Node.js + Prisma)\n- **Admin Panel** (Vite + React)\n- **Mobile App** (Expo EAS Build → Google Play + App Store)\n\n---\n\n## 📋 Prerequisites\n\n| Tool | Version | Notes |\n|------|---------|-------|\n| Node.js | 18+ | |\n| Docker + Docker Compose | Latest | For server deployment |\n| EAS CLI | Latest | `npm install -g eas-cli` |\n| Cloudinary Account | Free | For image uploads |\n| Resend Account | Free | For email (OTP) |\n\n---\n\n## 🗄️ Step 1 — Prepare Environment Variables\n\n### Root `.env` (Docker Compose)\n```env\nPOSTGRES_USER=elsayad\nPOSTGRES_PASSWORD=<strong_random_password>\nPOSTGRES_DB=ecommerce\nPOSTGRES_HOST=database\nPOSTGRES_PORT=5432\nDOCKER_DATABASE_URL=postgresql://elsayad:<password>@database:5432/ecommerce\nVITE_API_URL=https://api.yourdomain.com\n```\n\n### `backend/.env`\n```env\nDATABASE_URL=postgresql://elsayad:<password>@database:5432/ecommerce\nJWT_SECRET=<openssl rand -hex 64 output>\nJWT_EXPIRES_IN=7d\nNODE_ENV=production\nPORT=3000\nCORS_ORIGINS=https://admin.yourdomain.com,https://app.yourdomain.com\nEMAIL_PROVIDER=resend\nEMAIL_API_KEY=re_...\nEMAIL_FROM=noreply@yourdomain.com\nCLOUDINARY_CLOUD_NAME=your_cloud_name\nCLOUDINARY_API_KEY=your_api_key\nCLOUDINARY_API_SECRET=your_api_secret\nB2B_DISCOUNT_PERCENT=20\n```\n\n> **Generate a strong JWT_SECRET:**\n> ```bash\n> openssl rand -hex 64\n> ```\n\n---\n\n## 🐳 Step 2 — Deploy with Docker (Self-Hosted / VPS)\n\n```bash\n# Clone the repo on your server\ngit clone https://github.com/your-username/center-elsayad.git\ncd center-elsayad\n\n# Copy and configure .env files\ncp .env.example .env\ncp backend/.env.example backend/.env\n# Edit both files with your production values\n\n# Build and start production services\ndocker-compose -f docker-compose.prod.yml up -d --build\n\n# Check running services\ndocker-compose -f docker-compose.prod.yml ps\n\n# View logs\ndocker-compose -f docker-compose.prod.yml logs -f backend\n```\n\nThe production setup runs:\n- **Backend API** on port **3000**\n- **Admin Panel** (Nginx) on port **80**\n- **PostgreSQL** (internal, not exposed)\n\n---\n\n## ☁️ Step 3 — Deploy Backend to Railway / Render / Fly.io (Alternative)\n\n### Railway (Recommended for quick start)\n\n1. Go to [Railway](https://railway.app) → New Project → Deploy from GitHub\n2. Select the `backend` folder as your root\n3. Add environment variables from `backend/.env` in the Railway dashboard\n4. Railway will auto-detect the Dockerfile and deploy\n5. Add a **PostgreSQL** database plugin in Railway — it auto-sets `DATABASE_URL`\n\n### Render\n\n1. Go to [Render](https://render.com) → New Web Service → Connect GitHub\n2. Set **Root Directory** to `backend`\n3. Set **Build Command**: `npm install && npx prisma generate && npm run build`\n4. Set **Start Command**: `npx prisma migrate deploy && node dist/server.js`\n5. Add environment variables in the Render dashboard\n6. Add a **PostgreSQL** service and connect the URL\n\n---\n\n## 🌐 Step 4 — Deploy Admin Panel to Vercel / Netlify\n\n### Vercel\n```bash\nnpm install -g vercel\ncd admin-panel\nvercel --prod\n```\nSet the environment variable `VITE_API_URL=https://api.yourdomain.com` in the Vercel dashboard.\n\n### Netlify\n```bash\nnpm install -g netlify-cli\ncd admin-panel\nnpm run build\nnetlify deploy --prod --dir=dist\n```\n\nCreate `admin-panel/public/_redirects`:\n```\n/*  /index.html  200\n```\n\n---\n\n## 📱 Step 5 — Build Mobile App with EAS\n\n### Setup (First time only)\n```bash\n# Install EAS CLI\nnpm install -g eas-cli\n\n# Log in to your Expo account (create one at expo.dev)\neas login\n\n# Link project (creates EAS project ID)\ncd frontend\neas init\n\n# Paste the generated projectId into app.json → extra.eas.projectId\n```\n\n### Update API URL for production\nIn `frontend/src/services/` or wherever `apiClient` is configured:\n```typescript\n// Change from http://localhost:3000 to your production URL\nconst API_URL = process.env.EXPO_PUBLIC_API_URL || 'https://api.yourdomain.com';\n```\n\n### Build for both platforms\n```bash\n# Preview build (internal testing)\neas build --platform all --profile preview\n\n# Production build (store submission)\neas build --platform all --profile production\n```\n\n### Submit to stores\n```bash\n# Submit Android to Google Play\neas submit --platform android\n\n# Submit iOS to App Store\neas submit --platform ios\n```\n\n> **iOS Note:** You need an Apple Developer account ($99/year). You can build without a Mac using EAS cloud builds.\n\n---\n\n## ✅ Pre-Launch Checklist\n\n- [ ] Backend responds at production URL (`GET /health` returns 200)\n- [ ] Database migrations applied (`prisma migrate deploy`)\n- [ ] Seeder run for initial categories (`npm run db:seed`)\n- [ ] Admin panel loads and login works\n- [ ] Admin can create a product with an image\n- [ ] Mobile app connects to production API\n- [ ] Password reset email arrives in inbox\n- [ ] JWT_SECRET is strong and not the default\n- [ ] CORS_ORIGINS locked to production domains\n- [ ] Cloudinary credentials set\n- [ ] EAS build succeeds for Android\n- [ ] EAS build succeeds for iOS (needs Apple dev account)\n\n---\n\n## 🔐 Security Checklist\n\n- [ ] No secrets committed to git (check `.gitignore`)\n- [ ] Strong database password (random 32+ chars)\n- [ ] Strong JWT secret (`openssl rand -hex 64`)\n- [ ] `NODE_ENV=production` set\n- [ ] Rate limiting enabled (already done via `express-rate-limit`)\n- [ ] Helmet.js headers enabled (already done)\n- [ ] CORS restricted to known origins\n\n---\n\n## 🛠️ Useful Commands\n\n| Command | Location | Description |\n|---------|----------|-------------|\n| `docker-compose -f docker-compose.prod.yml up -d` | root | Start production stack |\n| `docker-compose -f docker-compose.prod.yml down` | root | Stop production stack |\n| `docker-compose -f docker-compose.prod.yml logs -f` | root | Tail all logs |\n| `npx prisma migrate deploy` | backend | Apply migrations in production |\n| `npm run db:seed` | backend | Seed initial data |\n| `eas build --platform all --profile preview` | frontend | Build test APK/IPA |\n| `eas build --platform all --profile production` | frontend | Build store release |\n
